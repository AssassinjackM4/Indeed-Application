🔗 **GitHub Repository:** [AssassinjackM4/Indeed-Application](https://github.com/AssassinjackM4/Indeed-Application.git)

## 📌 Overview

The **Indeed-Style Application** is a full-stack job portal designed to simplify the hiring process for both recruiters and candidates. It allows employers to post job openings, manage listings, and review candidate activity, while job seekers can browse available roles, search by relevant keywords, and apply to jobs through an intuitive interface.

This project demonstrates the integration of a **Spring Boot** backend with a **React** frontend using **RESTful APIs**, backed by **MongoDB** for flexible data storage and deployed online using **Vercel** for quick accessibility.

## 🧩 Project Architecture

The application follows a typical **client-server architecture**:

- **Frontend (React):** Handles user interactions, UI rendering, job search pages, forms, and application-related views.
- **Backend (Spring Boot):** Manages business logic, API endpoints, job posting operations, and application workflows.
- **Database (MongoDB):** Stores job listings, employer details, candidate information, and application records.
- **Deployment (Vercel):** Hosts the live project for public access.

## 🚀 Getting Started

### 1️⃣ Clone the repository
```bash
git clone https://github.com/AssassinjackM4/Indeed-Application.git
cd Indeed-Application
```

### 2️⃣ Set up the backend
- Install Java and your preferred IDE.
- Configure Spring Boot dependencies.
- Add MongoDB connection settings in the application configuration.
- Run the backend server.

### 3️⃣ Set up the frontend
```bash
cd frontend
npm install
npm run dev
```

### 4️⃣ Open the application
- Frontend should run locally in your browser.
- Backend APIs should connect with the frontend.
- MongoDB should be running or connected via cloud database.
